Associations de workflow

Cette section permet d'associer des workflows à des types d'entrée ou de commentaire (réponse). Lorsque vous associez un workflow à un type d'entrée ou de commentaire, le workflow est automatiquement démarré dès que l'entrée est créée ou le commentaire rédigé.

Association des workflows à des entrées

Cette section contient tous les types d'entrées que vous avez sélectionnés dans la section Types d'entrée par défaut de cette page. À côté de chaque type d'entrée dans la section Associations de workflow, Teaming affiche une liste déroulante contenant tous les processus de workflow que vous pouvez associer au type d'entrée.

Pour associer un workflow à un type d'entrée spécifique :

  1. Assurez-vous que vous avez autorisé la création de l'entrée dans ce dossier en sélectionnant le type d'entrée dans la section Types d'entrée par défaut sur cette page.

    Une fois que vous avez sélectionné un type d'entrée dans la section Types d'entrée par défaut et cliqué sur Appliquer, il apparaît dans la section Associations de workflow.

  2. Dans la section Associations de workflow, dans Types d'entrées, sélectionnez le workflow dans la liste déroulante en regard du type d'entrée auquel vous voulez appliquer ce workflow.

  3. Cliquez sur Appliquer > Fermer.

    Maintenant, dès qu'un utilisateur crée une entrée de ce type dans ce dossier, Teaming assigne automatiquement à l'entrée l'état initial du processus de workflow.

Lorsque vous créez un workflow dans un dossier, ce workflow est automatiquement ajouté à cette liste déroulante et peut être associé à des entrées de ce dossier. Pour plus d'informations sur le processus de création des workflows, reportez-vous à la section Creating and Managing Workflows (Création de workflows) du manuel Novell Teaming 2.1 Advanced User Guide (Guide de l'utilisateur avancé de Novell Teaming 2.1).

Association de workflows à des types de commentaire (réponse)

La section Types de commentaires ou de réponses permet d'assigner des workflows à démarrer lorsque les utilisateurs entrent un type de réponse spécifique.

Teaming affiche tous les types de réponse pour chaque type d'entrée par défaut qui figure dans la section Associations de workflow.

Lorsqu'un type d'entrée par défaut est également un type de réponse, il n'est pas à nouveau répertorié.

Pour plus d'informations, reportez-vous à la rubrique Mentions légales.

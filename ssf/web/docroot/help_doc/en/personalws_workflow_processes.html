Associating Workflow Processes

When you use the Workflow designer in the administrative portlet, the process is independent from any particular entry or folder. To test and use your workflow process, you can configure the folder to allow the use of the workflow in the folder. This means that you can view an entry and can use a menu item in the tool bar to start the process for that one entry.

Most production-ready systems require that a workflow process start automatically once a user creates an entry. To enable this, you can associate the workflow with a particular type of entry, as follows:

  1. View the folder that is to contain entries associated with the workflow process.

  2. Click Manage this folder > Configure.

    In the Workflow associations section of the form, Kablink Teaming lists default entry types next to drop-down lists.

  3. Select an entry type with which you want to associate a workflow process.

  4. In the drop-down list, select the name of the workflow process you want to associate with the default entry type.

  5. Click Apply.

Now, when someone creates an entry of that type in this folder, Kablink Teaming automatically places the entry in the initial state of the workflow process.

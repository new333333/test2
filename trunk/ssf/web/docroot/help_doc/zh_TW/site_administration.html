Kablink Teaming 站台管理

依預設，使用者便是其個人工作區 (及可能包含的資料夾) 及各自所建立之所有團隊工作區的管理員。如果您授權某人管理樹狀結構中的最上層工作區，則此人預設可以管理安裝中的所有工作區和資料夾 (因為所有工作區和資料夾均繼承該設定)。此人可以建立工作區和資料夾、設計項目和工作流程、建立群組、產生報告、重新編列位置的索引以提高搜尋效率等。

此外，站台管理員可以：

  • 設定電子郵件排程 (用於通知或張貼至 Teaming)

  • 設定角色定義 (存取控制)

  • 輸出及輸入定義 (請參閱有關組態的說明)

  • 輸入設定檔 (供開放原始碼客戶使用的 LDAP 輸入工具；請參閱安裝指南)

  • 使用工作區和資料夾範本 (請參閱有關組態的說明)

Teaming 管理 (尤其是組態設定) 分階段進行。是讓安裝實現最基本功能，還是等待使用者參與使貴組織對 Teaming 的使用達到最佳狀態，您可以在兩者之間加以平衡。

如需詳細資訊，請參閱《Kablink Teaming 2.0 管理指南》。

有關更詳細的資訊，請參閱 法律聲明.

Администрирование сайта Novell Teaming

По умолчанию пользователи являются администраторами своих личных рабочих областей (и папок, которые могут в них содержаться), а также любых создаваемых ими рабочих областей команд. Если Вы предоставляете кому-либо право на управление верхней рабочей областью в дереве, то по умолчанию этот пользователь может управлять всеми рабочими областями и папками в установке (поскольку все области и папки наследуют эту настройку). Этому пользователю разрешается создавать рабочие области и папки, конструировать записи и рабочие потоки, создавать команды, генерировать отчеты, проводить переиндексацию мест для повышения эффективности поиска и многое другое.

Кроме того, администраторы сайтов могут выполнять следующие действия.

  • Настраивать расписания электронной почты (для оповещений или для отправки в Teaming).

  • Настраивать определения ролей (управление доступом).

  • Экспортировать и импортировать определения (о настройке см. в справке).

  • Импортировать профили (средство импорта LDAP для клиентов ПО с открытым исходным кодом; см. руководство по установке).

  • Работать с шаблонами рабочих областей и папок (о настройке см. в справке).

Администрирование Teaming, в особенности настройка, выполняется по этапам. Вы можете уравновесить потребность обеспечения минимальной функциональности установки и необходимости ждать, пока участие пользователей обеспечит оптимальное использование Teaming в Вашей организации.

Дополнительные сведения см. в документе Руководство по администрированию Kablink Teaming 2.1.

Дополнительную информацию см. в Юридическая информация.

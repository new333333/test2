Allowing Non-Team Members to be Visitors

In addition to participating team members, you might want to allow other people to visit your team workspace in a more limited capacity. For example, you can add people to the Visitor role, which restricts them to reading or commenting on existing entries.

To grant visitor access to people:

  1. View your team workspace.

  2. Click Manage > Access control. Kablink Teaming displays the Configure access control page.

  3. In the This folder inherits... section, be sure that the No radio button is selected. If it is not, select the radio button, and then click the Apply button.

  4. Click the Add a user column header, type the first few letters of person's first or last name, and then, in the drop-down list, click the linked name. Kablink Teaming adds a row to the table for that person.

  5. In newly created row, select the box in the Visitor column. (If the Visitor column does not exist, click Add a role > Visitor, located toward the top of the table.)

  6. Repeat the last two steps until you have added all desired visitors.

  7. Click the Save changes button.

  8. Click Close.

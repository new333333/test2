Kablink Teaming Site Administration

By default, users are the administrators of their personal workspaces (and folders they might contain) and of any team workspaces they create. If you grant someone the right to manage the top workspace in the tree, then, by default, that person can manage all workspaces and folders in the installation (because all workspaces and folders inherit the setting). This person is allowed to create workspaces and folders, design entries and workflows, create groups, generate reports, re-index places to improve search efficiency, and more.

In addition, site administrators can:

  • Configure the e-mail schedules (for notifications or for posting into Teaming)

  • Configure role definitions (access control)

  • Export and import definitions (see Help about configuration)

  • Import profiles (an LDAP import facility for open-source customers; see the installation guide)

  • Work with workspace and folder templates (see Help about configuration)

Teaming administration, especially configuration, happens in stages. You can balance the need to make your installation minimally functional and the need to wait until user participation directs the optimal use of Teaming for your organization.

For more information, see the Kablink Teaming 2.0 Administration Guide.

Workflow Associations

This section enables you to associate workflows with entry types or with comment (reply) types. When you associate a workflow with an entry type or a comment type, the workflow is automatically started as soon as the entry is created or the comment is made.

Associating Workflows With Entries

This section contains all of the entry types that you selected in the Default Entry Types section on this page. Next to each entry type in the Workflow Associations section, Teaming displays a drop-down list containing all of the workflow processes that you can associate with the entry type.

To associate a workflow with a specific type of entry:

  1. Ensure that you have enabled the entry to be created in this folder, by selecting the entry type in the Default Entry Types section on this page.

    After you select an entry type in the Default Entry Types section and click Apply, it appears in the Workflow Associations section.

  2. In the Workflow Associations section, in the Entry Types section, select the workflow from the drop-down list next to the entry type where you want to apply the workflow.

  3. Click Apply > Close.

    Now, when someone creates an entry of that type in this folder, Teaming automatically places the entry in the initial state of the workflow process.

When you create a workflow in a folder, that workflow is automatically added to this drop-down list, then you can associate it to entries in this folder. For information on how to create workflows, see Creating Workflows in the Kablink Teaming 2.0 Advanced User Guide.

Associating Workflows With Comment (Reply) Types

The Comment or Reply Types section enables you to assign workflows to start when users enter a specific reply type.

Teaming displays all of the reply types for each default entry type that is listed in the Workflow Associations section.

When a default entry type is also a reply type, it is not listed again.

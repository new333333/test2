Understanding Default Users and Groups

Here are the default users and groups located in the Configure access control tables:

  • Owner of workspace or folder: The person who created it, or the person assigned to be the owner.

  • Team members: Those designated the status of team member in a given place, using the Team > Add or delete team members menu item.

  • All users: Every person registered with Kablink Teaming.

  • Site administrator: Person designated in a file on the server to be able to perform any task in any workspace or folder.

Here are the default roles:

  • Visitor: Has read-only and comment-only access.

  • Participant: Can create and modify entries (plus perform visitor tasks).

  • Team member: Is a Participant with a few administrative privileges.

  • Workspace and folder administration: Can create, modify, or delete workspaces or folders; moderate participation (modify or delete the entries of others, and so on); design entries and workflows; and more (plus perform Participant tasks).

  • Site administrator: Can use all designers (which create definitions), can create templates, and more (plus perform tasks from all other roles).

  • Workspace creator: Can create sub-workspaces (in root team workspaces, allows users to create their own team workspaces).

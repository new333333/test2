Configuring Access Control

Use this page to change the access control settings for this workspace or folder.

By default, workspaces and folders inherit access control settings from their parents. Use this page only if you want different access settings.

Before you can make any changes to the access control table, you must first ensure that the folder or workspace is not inheriting the users and roles from the parent directory. For information on how to do this, see Inherit Role Membership.

To see more detailed access control information than is contained in this section and the following sub-sections, see Access Control Overview.

The following sections describe how to modify the access control table to suit your access needs.

Adding Users and Groups

  1. Click either the Add a Group or Add a User link in the left column of the table.

  2. In the provided field, type the first few letters of the userâ€™s name or the group, and then, in the drop-down list, click the linked name.

    NovellKablink Teaming adds a row to the table for that user or group.

Viewing Role Tasks

In the access control table, role names serve as column headers.

  1. Click a column header (Participant, Team Member, and so on) to see a list of rights associated with that role.

Changing the Roles of Users or Groups

A role is a set of privileges; when you assign someone to a role, they automatically have all the rights associated with that role.

  1. Select and unselect check boxes to adjust who can function in which roles.

  2. Click OK.

Adjusting Access Controls in Another Place

  1. Go to the workspace tree at the top of this page, navigate to the place where you want to adjust the access controls, then click the link of that place in the tree.

    The Access Control page for that place appears.

Kablink Teaming 站点管理

默认情况下，用户是其个人工作空间（和工作空间可能包含的文件夹）以及各自创建的任何团队工作空间的管理员。如果您授予某人管理树的顶部工作空间的权限，则默认情况下，此人将可以管理 Teaming 中的所有工作空间和文件夹（因为所有工作空间和文件夹都继承设置）。这将允许此人创建工作空间和文件夹、设计条目和工作流程、创建组、生成报告、重索引位置以提高搜索效率等。

此外，站点管理员还可以：

  • 配置电子邮件日程表（用于通知或向 Teaming 投递邮件）

  • 配置角色定义（访问控制）

  • 导出和导入定义（请参见有关配置的帮助）

  • 导入配置文件（一项针对开放源代码客户的 LDAP 导入功能；请参见安装指南）

  • 使用工作空间和文件夹模板（请参见有关配置的帮助）

Teaming 管理（尤其是配置）要分阶段进行。您可以平衡以下两种需求：使安装实现的功能最少化的需求，以及等待用户参与以使组织实现对 Teaming 的最佳使用的需求。

有关更多信息，请参见《Kablink Teaming 2.0 管理指南》。
